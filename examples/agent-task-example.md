# Agent Task Example

## Role

Scoped implementation assistant

## Goal

Clarify one presentation-layer boundary without widening scope.

## verified_facts

- the task stays inside one or two files
- backend and deployment are out of scope

## confirmed_decisions

- do not widen scope
- do not rewrite confirmed product positioning

## risk_boundaries

- stop if provider, `.env`, deployment, state, memory, or gateway becomes relevant

## open_questions

- is one file enough for the requested clarification?
