# v1.0.0 Delta Handoff RC Candidate

Date: 2026-06-07

This document records the local release-candidate hardening state for the
BaseBrief v1.0 Delta Handoff line.

It is not a push, tag, release, package publication, provider integration, or
schema-v2 claim.

## Summary

BaseBrief v1.0 focuses on one narrow capability:

```text
Reviewable Delta Handoff Compiler
```

The v1.0 line adds a local `delta` command that turns current Project State,
git range facts, changed-file facts, and Seal/Diff state changes into a
reviewable `delta-handoff.md`.

The command is exposed through CLI Lite:

```text
node scripts/basebrief.js delta --repo <target-repo> --output-dir <dir> [--since <commit>] [--advance-baseline] [--json]
```

## Local Baseline Boundary

`delta-baseline.json` is a local delta baseline only:

```text
.basebrief/delta-baseline.json
```

It is not public history, not a replacement for Project State, and not a
schema-breaking change. `basebrief-project-state-v1` remains unchanged.

By default, `delta` does not advance the baseline. `--advance-baseline` is the
explicit local opt-in for writing the next delta baseline.

The follow-up baseline-advance closure is recorded in:

```text
docs/dogfooding/delta-handoff-baseline-advance-v1.0.md
```

That record keeps the same local-only boundary: `.basebrief/delta-baseline.json`
is ignored, not public history, and not a schema-breaking state change.
