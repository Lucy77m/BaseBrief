# v2.8.0 Continuation Harness Lite Local Closeout

Date: 2026-06-10

Status: local closeout

`v2.8.0` implements Continuation Harness Lite, the first narrow step from
individual Context Pack commands toward a reviewable continuation package. It
adds one local CLI command:

```text
node scripts/basebrief.js continue --repo <target-repo> --output-dir <dir> [--since <commit>] [--max-files <n>] [--json]
```

The command runs the existing `context-pack -> check -> resume` path and writes
a local package that a user can review before copying the starter into a new AI
window.

## What Changed

- Adds `scripts/basebrief_continuation_harness.js` as a thin orchestration
  layer.
- Exposes `continue` through CLI Lite help, routing, human output, and JSON
  output.
- Writes `CONTINUATION_REPORT.md`, `CHECK_SUMMARY.md`,
  `NEXT_WINDOW_STARTER.md`, `continuation.meta.json`, and `context-pack/`.
- Marks clean packages as `ready_for_receiver`, dirty or warning-only packages
  as `needs_review`, and errored packages as `blocked`.
- Keeps the top-level copied starter public-safe by normalizing the generated
  Context Pack path to `context-pack`.
- Adds independent tests in `tests/continuation-harness.test.js`.
- Records self-dogfooding evidence in
  `docs/dogfooding/continuation-harness-lite-v2.8.0.md`.

## JSON Boundary

`continue --json` reports metadata, output file paths, step status, git summary,
and check summary. It does not include the full resume prompt body and does not
add human-only `next_step` fields.

Existing Context Pack Check, Resume, Export, and Doctor JSON contracts remain
unchanged.

## Boundaries

- No provider request.
- No runtime integration.
- No plugin.
- No MCP server.
- No MCP tools.
- No schema-v2.
- No Workflow Runner.
- No Doctor expansion.
- No Export expansion.
- No automatic project task implementation.
- No automatic commit, push, tag, release, pull request, npm publish, or global
  CLI install.
- No Context Pack seven-file structure change.
- No raw private output, private absolute paths, `.env`, secrets, tokens,
  credentials, API keys, or bearer strings.
