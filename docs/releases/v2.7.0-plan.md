# v2.7.0 Context Pack Human Next-Step Hints Plan

Date: 2026-06-10

Status: v2.7-A human-output planning

This document opens the v2.7 line after the v2.6.x local closeout review
packet. It plans a narrow human-output usability improvement only. It does not
add a command, change package scripts, change JSON contracts, change Context
Pack files, publish, push, tag, release, open a pull request, or start Workflow
Runner work.

## Goal

BaseBrief v2.7 adds concise next-step hints to existing non-JSON CLI output so
a user who runs the current Context Pack commands can see the next recommended
local command without reading historical docs first.

The first target is the existing CLI Lite human output for:

```text
node scripts/basebrief.js context-pack --repo <target-repo> --output-dir <dir>
node scripts/basebrief.js check --input <context-pack-dir>
node scripts/basebrief.js export --input <context-pack-dir> --output-dir <dir>
node scripts/basebrief.js doctor --repo <target-repo> --context-pack <context-pack-dir>
```

## Planned Behavior

The hints are guidance text only:

- `context-pack` success should point to `check --input <context-pack-dir>`.
- `check` pass with zero errors should point to `resume --input
  <context-pack-dir>` and, when live repo comparison matters, `doctor --repo
  <target-repo> --context-pack <context-pack-dir>`.
- `check` warnings should keep warning exit semantics and suggest reviewing
  findings before `resume`.
- `check` errors should suggest fixing the pack before `resume`, `doctor`, or
  `export`.
- `export` success should point to `check --input <export-dir>` as the local
  verification step.
- `doctor` warning or error output should point back to live repo recheck,
  stale-pack review, or pack repair, depending on findings already reported.

## Contract Direction

The v2.7-A contract is:

- affect human output only
- keep `--json` output byte-shape-compatible for existing callers
- keep command exit semantics unchanged
- keep Context Pack Check, Resume, Export, and Doctor JSON contracts unchanged
- keep Context Pack Lite's seven-file structure unchanged
- reuse existing command results rather than adding a workflow runner or command
  chaining behavior

## Validation Gate

The local validation gate for this planning line is:

```text
npm run release-check
npm test
git diff --check
```

Implementation should additionally verify that non-JSON output contains the new
next-step hints while `--json` output remains unchanged for the same command
results.

When provider environment variables are absent, release checks must keep:

```text
provider_probe_status=skipped
```

## Rollback

Because the first implementation is expected to touch human formatting,
tests, docs, and release-check assertions only, rollback should be a small
revert of formatter text plus the matching tests and docs. If implementation
would require JSON, command, or contract changes, stop and keep v2.7 as a plan
only.
