# BaseBrief v0.5.1 Review Draft Gate Candidate

Date: 2026-06-05

`v0.5.1` adds `review-draft`, a local review gate that can turn a reviewed
`receiver-flow --guided` draft into a receiver-ready Markdown handoff. The gate
does not infer human intent. It only checks explicit text markers and checklist
state.

## Delivered Scope

- Added `node scripts/basebrief.js review-draft --draft <draft-context.md> --output <receiver-ready.md> [--json]`.
- The input must contain `handoff_status: draft_needs_review`.
- The input must contain the six guided human fields: `current_goal`,
  `verified_facts`, `confirmed_decisions`, `risk_boundaries`,
  `receiver_entry_task`, and `open_questions`.
- The input is rejected when it still contains `[EMPTY]`, `[NEEDS_REVIEW]`, or
  `[CANDIDATE]`.
- The review checklist must contain `- [x] <field> reviewed` for each required
  field.
- The output uses `handoff_status: ready_for_receiver` and
  `handoff_protocol_version: receiver-ready-v1`.
- The output file must be explicit, new, Markdown, and outside `.env` / `.git`
  paths.

## Compatibility

- Default `receiver-flow` behavior remains unchanged.
- `receiver-flow --guided` still writes draft-only output.
- BB9 handoff schema is unchanged.
- Receiver Safe Check config and result schemas are unchanged.
- CLI Lite remains local and zero-dependency.
- `package.json` remains private and validation-only.
- `provider_probe_status=skipped` remains the expected release-check result when
  provider env vars are absent.
