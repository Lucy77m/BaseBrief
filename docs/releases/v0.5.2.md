# BaseBrief v0.5.2 Receiver Flow Extract Candidate

Date: 2026-06-05

`v0.5.2` adds `receiver-flow --extract`, a local candidate extraction mode for
turning an explicit Markdown draft or context file into review-only receiver
flow candidate fields.

## Delivered Scope

- Added `node scripts/basebrief.js receiver-flow --repo <target-repo> --output-dir <dir> --extract --source <draft-or-context.md> [--json]`.
- `--source` is required and must point to an explicit local Markdown file.
- The command reads only the explicit source file and the explicit target repo
  state needed by Receiver Flow Draft.
- The command writes `flow-summary.json`, `receiver-check.json`,
  `draft-context.md`, and `extract-candidates.json`.
- Extracted fields are written as `[CANDIDATE]`.
- Missing extract fields are written as `[NEEDS_REVIEW]`.
- `draft-context.md` remains `handoff_status: draft_needs_review`.
- `review-draft` continues to reject extracted drafts until a human removes
  blocked markers and checks the review checklist.

## Compatibility

- Default `receiver-flow` behavior remains unchanged.
- `receiver-flow --guided` behavior remains unchanged.
- `review-draft` behavior remains unchanged.
- BB9 handoff schema is unchanged.
- Receiver Safe Check config and result schemas are unchanged.
- CLI Lite remains local and zero-dependency.
- `package.json` remains private and validation-only.
- `provider_probe_status=skipped` remains the expected release-check result when
  provider env vars are absent.
