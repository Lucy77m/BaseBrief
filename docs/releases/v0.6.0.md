# v0.6.0 Project State Directory Release

`v0.6.0` is the formal project-state directory release. It adds a local
`.basebrief/state.json` project state file and keeps BaseBrief as a local
handoff toolchain.

This release is not Auto Flow, not a hosted platform, not a provider gateway,
not a published npm package, and not a global CLI.

## Delivered

- New CLI Lite commands:
  - `state-init --repo <target-repo> --source <receiver-ready.md> [--json]`
  - `state-read --repo <target-repo> [--json]`
- New schema: `basebrief-project-state-v1`.
- New public docs: `docs/project-state.md`.
- New public-safe example: `examples/project-state/state.json`.
- Release checks cover the project-state docs, schema, CLI help, CLI smoke, and
  Artifact Checker safety pass.
- Release tag: `v0.6.0`.
- Release commit: `417541a`.

## Required Source

`state-init` only accepts a reviewed source with:

```text
handoff_status: ready_for_receiver
```

It does not accept `draft_needs_review` as project state.

## Boundaries

- No provider request.
- No Auto Flow.
- No receiver thread creation.
- No secret storage.
- No automatic `ready_for_receiver` promotion.
- No hosted service.
- No provider gateway.
- No published npm package.
- No global CLI installation.
- BB9 handoff schema is unchanged.
- Receiver Safe Check config and result schemas are unchanged.
- `provider_probe_status=skipped` remains the expected no-provider release gate.

## Validation

Expected local gate:

```bash
git diff --check
node --test tests/basebrief.test.js
node scripts/run_release_checks.js
npm run check
```

No provider environment is required.
