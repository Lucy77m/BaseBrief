# v2.1.0 Context Pack Check Plan

Date: 2026-06-08

This document freezes the v2.1-A check contract for BaseBrief Context Pack
Lite. It is a planning and contract record, not a checker implementation,
not a new command, not a JSON wire-shape change, and not a release.

## Goal

v2.1 Context Pack Check should make a generated Context Pack Lite directory
reviewable before it is handed to a fresh AI window.

The narrow goal is to verify that the pack is complete enough, public-safe,
bounded, and honest about missing inputs. The check must not prove that every
repo fact is true. It should only verify that the pack exposes the right facts,
gaps, boundaries, and review metadata for a receiver to inspect.

## Preferred Surface

Prefer the existing Artifact Checker surface:

```text
node scripts/basebrief.js check --input <context-pack-dir> --json
```

Do not add a new top-level command in v2.1-A. A future implementation may add
context-pack-specific detection under the existing `check` command, but this
contract does not change the current CLI help, JSON result shape, or
`context-pack` generator output.

## Contract Scope

Context Pack Check should verify these properties when implemented:

- the seven expected artifacts exist:
  `MANIFEST.md`, `REPO_MAP.md`, `KEY_FILES.md`, `RECENT_DELTA.md`,
  `RISK_BOUNDARIES.md`, `RECEIVER_STATE.md`, and `NEXT_WINDOW_STARTER.md`
- each artifact exposes shared review metadata:
  `Review status`, `Source`, `Trust`, and `Stale`
- `MANIFEST.md` includes branch, HEAD, worktree status, reading order, and
  safety notes
- `RISK_BOUNDARIES.md` preserves no provider request, no runtime integration,
  no plugin/MCP/IDE/hosted/cloud-memory work, no schema-v2, no repo dump, and
  no secrets / `.env` / token / credentials boundaries
- `RECEIVER_STATE.md` allows `not_available`, `not_applicable`, and
  `needs-review` without inventing receiver history
- `NEXT_WINDOW_STARTER.md` requires live repo fact recheck, read order,
  reporting gaps before edits, and keeping the v1.x Delta Handoff / Receiver
  line frozen unless explicitly reopened
- public-safety checks reject private absolute paths, secret-like strings, raw
  private output, and provider traces
- conservative thickness limits are documented before complex token estimation
  is added
- stale semantics remain visible through `stale`, `generated`, and
  `needs-review`

## Non-Goals

v2.1-A does not add:

- provider request
- runtime integration
- plugin, MCP server, IDE extension, hosted service, or cloud-memory behavior
- Workflow Runner
- schema-v2
- `basebrief-project-state-v2`
- `basebrief-sidecar-v2`
- new `context-pack` JSON wire shape
- AI automatic summary
- vector database, embedding, or semantic index
- watcher, daemon, dashboard, or prediction engine
- push, tag, release, pull request, npm publish, or global CLI install

## Fixture Direction

The future v2.1-B implementation should use small fixtures rather than raw
private generated output:

- a clean public-safe Context Pack Lite directory
- a broken pack missing one required artifact
- a pack missing shared review metadata
- a pack with unsafe private-path or secret-like content
- a pack with over-thick files once conservative limits are implemented
- a pack that uses `not_available` / `not_applicable` / `needs-review`
  correctly

Fixtures should live in public example or test fixture paths only after they
have been normalized and checked for secrets and private paths.

## Validation Gate

The local validation gate for this planning slice is:

```text
node scripts/run_release_checks.js
npm run check
git diff --check
```

Expected release-check output without provider env remains:

```text
provider_probe_status=skipped
```

## Next Slice

The next narrow implementation candidate is v2.1-B Minimal Checker
Integration:

- detect a Context Pack Lite directory under `check --input <dir>`
- add context-pack-specific rule families without changing unrelated checker
  behavior
- keep the checker read-only
- keep JSON output compatible with the current artifact checker result shape
- add clean and broken fixtures only after the contract is stable
