# v2.9.1 Project Profile CWD Hint Polish

Date: 2026-06-10

Status: local closeout

`v2.9.1` is a small dogfooding fix on top of `v2.9.0` Project Profile /
Recipes Lite. A real local run showed that placing a generated profile outside
the repository could make `continue --profile` resolve the public-safe
`repo_hint` relative to the profile file instead of the command's current
working directory.

## What Changed

- Keeps the `basebrief-project-profile-v1` contract unchanged.
- Keeps profile JSON public-safe and still avoids private absolute paths.
- Lets `continue --profile` resolve `repo_hint` in this order:
  profile directory, current working directory, then current working directory
  when its basename matches the hint.
- Adds a regression test for profile files stored outside the target repo while
  running the command from the repo root.

## Dogfooding Evidence

The trigger scenario was:

```text
profile-init --repo . --output <temp>/basebrief-profile.json
continue --profile <temp>/basebrief-profile.json --output-dir <temp>/continue-profile
```

Before the fix, `continue --profile` could look for a sibling directory under
the profile output directory. After the fix, the same public-safe repo hint can
resolve against the command's current working directory.
