# v2.10.0 Adoption Polish / Public Freeze

Date: 2026-06-10

Status: local closeout candidate

v2.10.0 closes the current public BaseBrief expansion loop by making the first-run story smaller and clearer. It is not a new feature line.

## Goal

Make BaseBrief understandable to a new user in a few minutes:

```text
one local command -> one continuation package -> copy NEXT_WINDOW_STARTER.md
```

The public positioning is:

```text
BaseBrief is a local-first AI project continuation tool.
```

It should not be presented as an Agent runtime, workflow platform, provider gateway, hosted service, project manager, or full spec framework.

## Changes

- README and README.en now lead with the simple continuation path.
- `docs/why-basebrief.md` explains the problem without architecture-heavy language.
- `docs/concepts-simple.md` explains the few concepts a first-time user needs.
- `docs/advanced.md` moves advanced commands out of the first-run path.
- `examples/simple-continuation/` gives a small public-safe Todo-app handoff example.

## Public Path

The first command remains:

```text
node scripts/basebrief.js continue --repo . --output-dir tests/outputs/private/continue
```

The first file to copy remains:

```text
NEXT_WINDOW_STARTER.md
```

## Advanced Scope

These remain available but are not first-run selling points:

- Context Pack / Check / Resume
- Project Profile
- Workflow Runner Lite
- Export and Doctor
- Seal/Diff, Delta, Project State
- Receiver and Sidecar tools
- cache-ready experiments

## Freeze Boundary

After this polish line, BaseBrief should prefer maintenance over expansion:

- fix bugs
- repair misleading docs
- preserve public-safe output
- keep tests and release checks green
- keep provider/cost language conservative

Do not add MCP, provider/runtime integration, daemon/watchers, schema-v2, automatic project-task execution, automatic commit/push/tag/release/PR, npm publishing, global install flow, or GUI/platform work without a new explicit decision.
