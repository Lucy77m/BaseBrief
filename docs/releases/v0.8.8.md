# v0.8.8 Starter Language Routing Candidate

`v0.8.8` is a Sidecar usability patch for the copyable
`new-window-starter.md` artifact. It adds `--starter-language auto|zh-CN|en|ja`
to `sidecar-build` so the starter shell can match the user's language while the
receiver contract remains unchanged.

## Changes

- `sidecar-build` accepts `--starter-language auto|zh-CN|en|ja`; the JS API
  accepts `buildSidecarBundle({ starterLanguage })`.
- `auto` detects Chinese, English, and Japanese from the natural-language
  handoff body and falls back to `zh-CN` when mixed or unclear.
- Explicit `zh-CN`, `en`, and `ja` starter language packs localize the
  user-facing shell only.
- Protocol fields, file names, paths, schema names, and English hard-stop
  anchors stay literal, including `current_goal`, `receiver_entry_task`,
  `risk_boundaries`, No provider request, No raw private output, No runtime
  integration, No schema change, and No auto-advance.
- Localized first-response instructions preserve `pass/fail` as the receiver
  acceptance report anchor.
- `sidecar-check` accepts localized starter shells while preserving the same
  required starter contract and old v0.8 bundle compatibility.

## Boundaries

- No provider request.
- No raw private output.
- No runtime integration.
- No schema change.
- No Auto Flow.
- No clipboard automation.
- No UI integration.
- No automatic translation service.

`sidecar-build` remains a local Project State consumer for
`basebrief-project-state-v1`. `sidecar-check` remains a read-only structure and
public-safety gate for `basebrief-sidecar-v1`.

## Release Gate

The no-provider release gate remains:

```text
provider_probe_status=skipped
```
