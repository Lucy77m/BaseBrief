# v0.8.5 Manual Receiver Smoke Result Intake Candidate

`v0.8.5` is a documentation and release-check hardening checkpoint for manual
receiver smoke result intake. It does not add a new CLI command, API, schema,
provider smoke, runtime integration, tag, release, or npm publish step.

## What Changed

- Added a public-safe intake format for manual OpenCode and Claude Code
  receiver smoke summaries.
- Kept all current receiver smoke rows at `not_run` with `manual_required`
  notes until user-approved external evidence is supplied.
- Documented the required acceptance fields before any future result may be
  marked `passed`.
- Added release-check and unit-test coverage to keep the manual status honest.

## Result

- OpenCode generic receiver smoke: not_run, manual_required.
- Claude Code generic receiver smoke: not_run, manual_required.
- OpenCode openclaw receiver smoke: not_run, manual_required.
- Claude Code openclaw receiver smoke: not_run, manual_required.
- No external receiver prompt was invoked from Codex.
- No sidecar bundle was regenerated or committed.

## Boundaries

- No provider request.
- No raw private output.
- No runtime integration.
- No schema change.
- No Auto Flow.
- No receiver thread creation from Codex.
- Wait for user confirmation before continuing from a generated bundle.
- No OpenClaw/Hermes runtime connection.
- No OpenClaw/Hermes profile/config/memory/workspace writes.
- No push, tag, release, or npm publish in this checkpoint.

A future `passed` result requires a public-safe acceptance summary covering
BaseBrief, v0.8.x, current commit, `current_goal`, receiver entry task, at
least two risk boundaries, wait for user confirmation, no auto-advance, no
provider, and no runtime.

`sidecar-build` remains a local Project State consumer for
`basebrief-project-state-v1`. `sidecar-check` remains a read-only structure and
public-safety gate for `basebrief-sidecar-v1` bundles.
