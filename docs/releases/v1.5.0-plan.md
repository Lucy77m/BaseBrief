# v1.5.0 Delta Receiver Lint Mini Plan

Date: 2026-06-07

This document records the recommended v1.5.0 planning direction after the
v1.4 Delta Receiver Usage Pack line became locally closeout-ready.

It is a planning baseline, not a push, tag, release, package publication,
provider integration, runtime integration, plugin, MCP, IDE, hosted service,
Auto Flow behavior, global CLI install, npm publication, command output format
change, JSON schema claim, or schema-v2 claim.

## Summary

BaseBrief v1.5.0 should keep the Delta Receiver line local-first and
review-based by connecting the v1.2-v1.4 receiver report discipline to the
existing artifact checker:

```text
Delta Receiver Lint Mini
```

The goal is not to create a new runtime, command, or schema. The goal is to
make public-safe receiver reports harder to copy incorrectly by checking
explicit receiver Markdown and receiver result JSON shapes.

## Recommended Direction

Prefer a checker/docs/examples/tests slice:

- extend the artifact checker with receiver-specific rule families
- detect receiver artifacts by explicit receiver shape, not broad repository
  keyword search
- treat missing machine fields, missing required report sections, missing
  starter `pass/fail`, missing `wait for user confirmation`, missing fact-layer
  separation, and contradictory receiver result JSON as errors
- treat missing `difference_found` semantics and missing non-blocking
  historical `commits_in_range` drift explanation as warnings
- cover both Markdown/text receiver reports and
  `basebrief-receiver-check-result-v1` JSON outputs

## v1.5.0 In Scope

- Artifact checker receiver lint for explicit receiver Markdown and result JSON.
- Public docs and examples updated to describe and satisfy the new checker.
- Release-check coverage and focused tests for the new rule family.

## v1.5.0 Out Of Scope

- No provider request.
- No runtime integration.
- No plugin, MCP, IDE, hosted, or cloud-memory work.
- No schema-v2 work.
- No `basebrief-project-state-v2`.
- No `basebrief-sidecar-v2`.
- No new CLI command.
- No new npm script.
- No machine-readable JSON schema.
- No command output format change.
- No Auto Flow.
- No push, tag, release, pull request, npm publish, or global CLI install.
- No raw private output in public docs.

## Success Criteria

v1.5.0 is successful if all of the following are true:

1. `basebrief check` can fail explicit receiver artifacts that are missing core
   receiver contract pieces.
2. Receiver lint only triggers on explicit receiver Markdown or
   `basebrief-receiver-check-result-v1` JSON shapes; ordinary docs that merely
   mention field names are not treated as receiver reports.
3. Existing receiver semantics stay stable: `difference_found` remains a
   completed verification result, not an agent failure.
4. No schema, command, runtime, provider, plugin, MCP, IDE, Auto Flow, hosted,
   or publication scope is introduced.
5. `provider_probe_status=skipped` remains the no-provider validation gate when
   provider environment variables are absent.

## Deferred Alternatives

Schema changes, machine-parsed receiver Markdown formats, runtime enforcement,
and repository-wide keyword-triggered receiver lint remain deferred until
repeated local usage shows a concrete blocker.

When provider environment variables are absent, release checks must keep:

```text
provider_probe_status=skipped
```
