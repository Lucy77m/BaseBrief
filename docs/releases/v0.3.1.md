# BaseBrief v0.3.1 Release Candidate

Status: local release candidate

Date: 2026-06-05

BaseBrief v0.3.1 closes the first Receiver Stabilization pass after v0.3.0. It keeps Receiver Safe Check v1, BB9, Full/Lite/cache-ready routing, provider profiles, and CLI Lite command semantics unchanged.

## Highlights

### Receiver Stabilization

- Added a public-safe receiver friction log for repeated handoff observations.
- Added receiver examples for `difference_found`, `blocked`, and `response_language: match_latest_user_message`.
- Clarified that `difference_found` is a completed receiver task, not an agent failure.
- Kept `blocked` reserved for unsafe or impossible receiver checks.

### Local Validation Scripts

- Added minimal local npm scripts:
  - `npm test`
  - `npm run release-check`
  - `npm run check`
- `package.json` stays `private: true`, has zero dependencies, and does not define `name`, `version`, `bin`, `publishConfig`, or package files.
- These scripts are validation shortcuts only. BaseBrief is not a published npm package, global CLI, plugin, or provider integration.

## Compatibility

- BB9 handoff schema is unchanged.
- Receiver Safe Check config and result schemas are unchanged.
- `scripts/basebrief.js` command behavior is unchanged.
- Existing Node commands remain the canonical command surface.
- Full, Lite, and cache-ready routing is unchanged.
- Auto Flow Skeleton is not introduced in v0.3.1; it remains a v0.3.2-or-later candidate that needs a separate plan.

## Evidence

Repository-reproducible validation for this release candidate includes:

- `git diff --check`
- `node --test tests/basebrief.test.js`
- `node scripts/run_release_checks.js`
- `npm run check`

The release check must preserve `provider_probe_status=skipped` when provider environment variables are absent.

No provider request is required or performed for this release-candidate preparation. No external receiver matrix, OpenCode smoke, Claude Code smoke, Auto Flow run, or provider benchmark is required for v0.3.1.

## Known Limits

- The receiver friction log is a public-safe summary, not a raw transcript archive.
- The receiver examples are static documentation examples, not new runtime behavior.
- `expected_changed_files` still does not prove file-content integrity.
- `generated_at` still does not create an automatic stale-handoff policy.
- BaseBrief still does not automatically switch receiver working directories.

## Release Gate

The local release candidate is ready only after:

- all automated tests and release checks pass
- public artifacts report zero Artifact Checker errors and warnings
- protected interfaces remain unchanged
- commits contain no secrets or private absolute paths
- `provider_probe_status=skipped` is reported when provider env vars are absent
- the user separately approves push, tag, and release

This document is not a release announcement. No push, tag, or formal release is performed by the release-candidate preparation task.
