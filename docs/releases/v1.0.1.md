# v1.0.1 Delta Receiver Clarity Patch

Date: 2026-06-07

This patch improves how receivers read `delta-handoff.md`.

It is not a provider request, runtime integration, plugin, MCP, IDE,
schema-v2, push, tag, release, npm publication, or platform claim.

## Summary

`v1.0.1` keeps the v1.0 Delta Handoff line narrow and local-first. It adds
receiver-facing explanation text to the generated Markdown handoff so a new
window can understand first-run and baseline-present states without guessing.

The patch does not change:

- `basebrief-delta-handoff-v1`
- `basebrief-delta-baseline-v1`
- `basebrief-project-state-v1`
- CLI JSON field names or JSON semantics

## Clarity Anchors

Generated `delta-handoff.md` now explains:

- `reviewed` sections come from the current reviewed Project State
- `needs-review` sections are generated git, worktree, and Seal/Diff summaries
- `baseline_source: missing` is normal on the first delta run
- `no-baseline..HEAD` is a human-readable sentinel, not a git revision range
- `commits_in_range: 0` does not mean the worktree is clean
- `stateDiff.status: unchanged` means reviewed Project State matches the delta
  baseline, not that git or worktree content is unchanged

The public-safe delta example now shows a baseline-present state where
`commits_in_range: 0`, `Worktree Changed Files` is non-empty, and
`stateDiff.status: unchanged`.
