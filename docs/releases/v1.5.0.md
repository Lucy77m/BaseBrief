## Receiver Lint Contract

The v1.5 line adds these receiver rule families:

- `receiver.missing-machine-field`
- `receiver.missing-report-section`
- `receiver.missing-human-anchor`
- `receiver.missing-fact-layer`
- `receiver.invalid-result-consistency`
- `receiver.missing-difference-semantics`
- `receiver.missing-drift-semantics`

Detection stays explicit:

- receiver result JSON must declare
  `schemaVersion: basebrief-receiver-check-result-v1`
- starter-style Markdown must expose the machine fields plus starter report
  sections such as `receiver_entry_task` and fact-layer sections
- delta-style Markdown must expose the machine fields plus the delta report
  sections such as `blocking_or_repair_notes`, `live_repo_state`, and
  `next_narrow_slice`
- ordinary docs do not become receiver artifacts only because they mention
  field names

## Severity

- errors: missing core machine fields, missing required receiver report
  sections, missing starter `pass/fail`, missing
  `wait for user confirmation`, missing fact-layer separation, or contradictory
  receiver result JSON
- warnings: missing `difference_found` semantics explanation or missing
  non-blocking historical `commits_in_range` drift explanation

`difference_found` remains a completed verification result, not an execution
failure.
