# v2.7.0 Context Pack Human Next-Step Hints Local Closeout

Date: 2026-06-10

Status: local closeout

`v2.7.0` closes the first Context Pack Human Next-Step Hints implementation
slice. It adds concise next-step hints to existing non-JSON CLI output only.
It does not add a command, change package scripts, change JSON contracts,
change Context Pack files, publish, push, tag, release, open a pull request, or
start Workflow Runner work.

## What Changed

- Adds `next_step=` guidance to human output for `context-pack`, `check`,
  `export`, and `doctor`.
- Adds `optional_next_step=` guidance for clean or warning-only `check` output
  when a user may continue with `resume` or compare against a live repository
  with `doctor`.
- Keeps `check` errors actionable by pointing users back to pack repair before
  `resume`, `doctor`, or `export`.
- Keeps `doctor` warning and error output focused on live repo recheck,
  stale-pack review, and pack repair.
- Adds tests that verify human output contains the new hints while `--json`
  output does not gain a `next_step` field.

## Human Output Examples

The new human-only hints use stable line prefixes:

```text
next_step=node scripts/basebrief.js check --input <context-pack-dir>
next_step=node scripts/basebrief.js resume --input <context-pack-dir>
optional_next_step=node scripts/basebrief.js doctor --repo <target-repo> --context-pack <context-pack-dir>
next_step=node scripts/basebrief.js check --input <export-dir>
next_step=recheck live repo facts and refresh the context pack if stale
next_step=repair the context pack before resume or export
```

These lines are guidance text, not machine-readable contracts.
