## Usage Pack Contract

The v1.4 usage pack keeps these machine fields stable:

- `receiver_task_status`
- `repository_state_status`
- `declared_checks_status`
- `handoff_acceptance`

It also keeps these human-facing anchors stable:

- `pass/fail`
- `wait for user confirmation`

The pack keeps source-window inherited facts, live repo facts, and
receiver-window rechecks separate before implementation resumes.

## Decision Routing

- use the Delta report kit for Delta continuity acceptance and live repo
  rechecks against `delta-handoff.md`
- use the golden-path starter report for starter-facing continuity replies that
  must keep `current_goal`, `receiver_entry_task`, `risk_boundaries`,
  `declared_checks_status`, `pass/fail`, and `wait for user confirmation`
- `difference_found` and human `fail` can coexist; that still means the
  receiver completed entry verification and accurately reported a mismatch
- `blocked` stays reserved for unsafe or impossible-to-complete verification
- historical `commits_in_range` drift remains non-blocking when refreshed
  branch, HEAD, and worktree facts still match live repository state
