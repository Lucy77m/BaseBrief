# v0.8.1 Sidecar Check Hardening Candidate

`v0.8.1` adds a local, read-only acceptance gate for sidecar bundles generated
from Project State.

```text
node scripts/basebrief.js sidecar-check --input <sidecar-dir> [--json]
```

## Scope

- Validate an existing sidecar bundle without creating or modifying files.
- Require `handoff.md`, `next-chat-prompt.md`, `receiver-entry-task.md`,
  `risk-boundaries.md`, `state-summary.json`, and `manifest.json`.
- Parse `manifest.json` and `state-summary.json`.
- Require `schemaVersion: basebrief-sidecar-v1`.
- Require `projectStateSchemaVersion: basebrief-project-state-v1`.
- Require `target` to be `generic` or `openclaw`.
- Require non-empty `current_goal` and `receiver_entry_task`.
- Require at least two `risk_boundaries`.
- Require `next-chat-prompt.md` to include the current goal, receiver task,
  at least two risk boundaries, wait for user confirmation, No provider
  request, No raw private output, No runtime integration, and No auto-advance.
- For `openclaw`, require explicit OpenClaw/Hermes runtime,
  profile/config/memory/workspace prohibition wording.

## Boundaries

- No provider request.
- No raw private output.
- No runtime integration.
- No schema change.
- No Auto Flow.
- No receiver thread creation.
- No push, tag, release, or npm publish in this checkpoint.

`sidecar-check` reuses the artifact checker so generated bundles fail if they
contain secret-like strings, private absolute paths, or raw provider output.
The command is a consumer of sidecar output; it does not mutate
`.basebrief/state.json`, `.basebrief/history/`, or `.basebrief/sidecar/`.
