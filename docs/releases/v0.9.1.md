# v0.9.1 Golden Path Closure Candidate

`v0.9.1` closes the public usability gap after `v0.9.0` Integrated Handoff
Readiness. It does not define a new feature line. It makes the existing local
handoff path easier to follow as one public golden path.

## Golden Path Goal

The target workflow remains:

```text
receiver-ready.md -> state-init/state-advance -> sidecar-build -> sidecar-check -> new-window-starter.md -> receiver first response
```

This candidate keeps the first-use and follow-up branches explicit:

- first local continuity pass uses `state-init`
- later reviewed handoff updates use `state-advance`
- `state-status`, `state-validate`, and `state-history` stay optional read-only
  checks

## What Changed

- Adds one public `Integrated Handoff Golden Path` guide.
- Re-links README, quickstart, Project State, CLI Lite, testing docs, and the
  docs index so the integrated local path no longer has to be reconstructed
  from scattered pages.
- Preserves the receiver first-response contract around
  `new-window-starter.md`: restate key fields, report `pass/fail`, and wait for
  user confirmation.
- Adds release-check and test coverage so future doc drift breaks locally
  instead of silently weakening the public path.

## Boundaries

- No provider request.
- No raw private output.
- No runtime integration.
- No schema change.
- No Auto Flow.
- No plugin or platform work.
- No published npm package or global CLI.
- No v1.0 claim.
- No expansion into `v0.9.2+` feature work from this patch.

## Release Gate

The no-provider release gate remains:

```text
provider_probe_status=skipped
```

Required local checks:

```text
node --test tests/basebrief.test.js
node scripts/run_release_checks.js
git diff --check
```
