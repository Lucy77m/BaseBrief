# v0.9.0 Integrated Handoff Readiness Candidate

`v0.9.0` defines the first integrated readiness line after the `v0.8.x`
Sidecar receiver closure. It is a public hardening candidate for the local
handoff chain, not a provider, runtime, schema, Auto Flow, platform, or v1.0
release.

## Readiness Goal

The target workflow is:

```text
receiver-ready handoff -> Project State -> Sidecar bundle -> receiver first response
```

This candidate keeps the existing local pieces aligned so users can understand
when to use `state-init`, `state-advance`, `sidecar-build`, `sidecar-check`,
and the copyable `new-window-starter.md` without treating BaseBrief as an
agent runtime.

## What Changed

- Records `v0.9.0` as Integrated Handoff Readiness / public hardening.
- Connects Project State, Sidecar, receiver examples, testing docs, and release
  checks as one local handoff-readiness line.
- Keeps the `pass/fail` receiver acceptance anchor from `new-window-starter.md`
  visible as the human first-response contract.
- Keeps OpenClaw/Hermes evidence scoped to public-safe manual first-response
  follow-up only.
- Updates roadmap wording so the current line reflects the completed `v0.8.x`
  Sidecar receiver closure instead of the older v0.3/v0.6 framing.

## Boundaries

- No provider request.
- No raw private output.
- No runtime integration.
- No schema change.
- No Auto Flow.
- No Web UI.
- No plugin or platform work.
- No published npm package or global CLI.
- No v1.0 claim.
- No cross-provider cache claim.
- No claim based on audited billing records.
- No push, tag, release, publish, or pull request in this local candidate.

## Release Gate

The no-provider release gate remains:

```text
provider_probe_status=skipped
```

Required local checks:

```text
node --test tests/basebrief.test.js
node scripts/run_release_checks.js
git diff --check
```
