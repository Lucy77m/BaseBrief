# v2.3.0 BaseBrief Format Plan

Date: 2026-06-08

Status: v2.3-A contract freeze

This document freezes the docs-first BaseBrief Format direction. It does not
implement a command, generator, JSON schema file, format emission, schema-v2,
provider request, runtime integration, plugin, MCP server, IDE extension,
hosted service, cloud-memory feature, or Workflow Runner.

## Goal

BaseBrief v2.3 defines a future local-first packaging contract for AI coding
agent handoffs after Context Pack Lite, Context Pack Check, and Resume have
stabilized.

The planned artifact family is:

```text
context-pack/
context-pack.md
context.json
```

The wording should stay restrained: BaseBrief Format is a local handoff format,
not an industry standard, universal protocol, hosted platform, runtime, or
agent framework.

## Contract Direction

The v2.3-A contract is:

- `context-pack/` remains the reviewable directory form for human inspection
- `context-pack.md` is a future single-file readable packaging form
- `context.json` is a future machine-readable summary form
- all generated or packaged facts must preserve source, trust, review status,
  stale, `not_available`, `not_applicable`, and `needs-review` semantics
- receivers must still recheck live repo facts before implementation

## Non-goals

- No command.
- No implementation.
- No generator.
- No JSON schema file.
- No schema-v2.
- No format emission.
- No provider request.
- No runtime integration.
- No plugin, MCP, IDE, hosted service, or cloud-memory behavior.
- No Workflow Runner.
- No change to `basebrief-project-state-v1`.
- No change to `basebrief-sidecar-v1`.
- No change to Context Pack Lite generator output.
- No change to `check --input <dir> --json` top-level shape.
- No push, tag, release, pull request, npm publish, or global CLI install.
- No raw private output, private absolute paths, `.env`, secrets, tokens,
  credentials, API keys, or bearer strings.

## Future Implementation Notes

A later v2.3-B implementation may add a local formatter only after this
contract is reviewed and dogfooding shows the format is useful.

That later implementation should start from existing checked Context Pack Lite
inputs. It should not invent missing facts, silently refresh inherited facts,
or collapse warning/error semantics.

## Validation Gate

The local validation gate for this planning line is:

```text
npm test
npm run release-check
git diff --check
```

When provider environment variables are absent, release checks must keep:

```text
provider_probe_status=skipped
```
