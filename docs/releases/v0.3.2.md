# BaseBrief v0.3.2 Release Candidate

Status: local release candidate

Date: 2026-06-05

BaseBrief v0.3.2 adds the Receiver Flow Draft Skeleton. It is a local draft helper for preparing receiver handoff material before it is reviewed and shared. It does not change Receiver Safe Check v1, BB9, Full/Lite/cache-ready routing, provider profiles, or existing CLI Lite command semantics.

## Highlights

### Receiver Flow Draft Skeleton

- Added `node scripts/basebrief.js receiver-flow --repo <target-repo> --output-dir <dir> [--json]`.
- Writes `flow-summary.json`, `receiver-check.json`, and `draft-context.md` to an explicit output directory.
- Collects only mechanical repository facts: branch, HEAD, changed files, generated time, and a state-only Receiver Safe Check config.
- Keeps every generated handoff draft in review state:

```text
handoff_status: draft_needs_review
```

### Safety Boundary

- No provider request.
- No receiver thread creation.
- No Auto Flow completion.
- No final handoff generation.
- No automatic promotion to `ready_for_receiver`.
- No tracked target-repository file writes.
- No overwrite of existing output files.

## Compatibility

- BB9 handoff schema is unchanged.
- Receiver Safe Check config and result schemas are unchanged.
- `receiver-init` and `receiver-check` behavior is unchanged.
- Existing CLI Lite commands remain available.
- `receiver-flow` is draft-only and must be reviewed before it can be turned into a receiver-ready handoff.
- Auto Flow remains out of scope and requires a separate future plan.

No provider request is required or performed for this release-candidate preparation. No external receiver matrix, OpenCode smoke, Claude Code smoke, Auto Flow run, Web UI check, Cursor adapter test, or provider benchmark is required for v0.3.2.

## Known Limits

- `receiver-flow` generates a draft, not a final handoff.
- The generated draft must be reviewed before sharing.
- The generated Receiver Safe Check config is state-only until a human adds optional declared checks.
- `expected_changed_files` still does not prove file-content integrity.
- `generated_at` still does not create an automatic stale-handoff policy.
