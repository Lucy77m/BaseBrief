# v2.5.0 Context Pack Doctor Plan

Date: 2026-06-08

Status: v2.5-A contract freeze

This document freezes the v2.5 Doctor direction. It does not implement
`status`, watcher behavior, daemon behavior, auto-fix, provider requests,
runtime integration, plugin work, MCP server/tools, IDE integration,
hosted/cloud-memory behavior, schema-v2, or Workflow Runner.

## Goal

BaseBrief v2.5 adds a manually triggered, read-only diagnostic command for a
target repository and an explicit Context Pack Lite directory:

```text
node scripts/basebrief.js doctor --repo <target-repo> --context-pack <context-pack-dir> [--json]
```

The command is conservative. It reports source-backed maintenance gaps, stale
snapshot indicators, checker errors or warnings, and local-only boundary
coverage. It does not predict correctness and does not replace the receiver's
live repo recheck.

## Contract Direction

The v2.5-A contract is:

- require both `--repo` and `--context-pack`
- do not auto-discover packs
- read only local repository facts and Context Pack Lite files
- do not write files, repair files, generate packs, or mutate Project State
- reuse `checkArtifacts({ inputPath })` for Context Pack validation
- report checker errors as doctor error findings without throwing away the
  rest of the diagnostic report
- keep JSON public-safe by reporting relative paths, short evidence text, and
  no private absolute paths

The JSON contract version is:

```text
basebrief-doctor-v1
```

## Finding Contract

Each finding uses:

```json
{
  "severity": "error|warning|info",
  "ruleId": "doctor.<rule>",
  "message": "...",
  "source": "git|context-pack|check|docs",
  "evidence": "public-safe short text"
}
```

The first rule set is:

- `doctor.worktree-dirty`
- `doctor.pack-head-stale`
- `doctor.pack-branch-mismatch`
- `doctor.pack-check-error`
- `doctor.pack-check-warning`
- `doctor.live-recheck-required`
- `doctor.no-provider-boundary`

## Boundaries

- No `status` command in v2.5.
- No watcher, daemon, or background monitor.
- No auto-fix or file writing.
- No Context Pack Lite generator output change.
- No `check --input <dir> --json` top-level shape change.
- No `resume --input <context-pack-dir>` behavior change.
- No `export --input <context-pack-dir> --output-dir <dir>` contract change.
- No provider request.
- No runtime integration.
- No plugin.
- No MCP server.
- No MCP tools.
- No IDE integration.
- No hosted service.
- No cloud-memory behavior.
- No schema-v2.
- No Workflow Runner.
- No push, tag, release, pull request, npm publish, or global CLI install.
- No raw private output, private absolute paths, `.env`, secrets, tokens,
  credentials, API keys, or bearer strings.

## Validation Gate

The local validation gate for this planning line is:

```text
node scripts/basebrief.js doctor --repo . --context-pack examples/context-pack-lite --json
node --test tests/basebrief.test.js --test-name-pattern "Doctor|v2.5|Context Pack"
npm test
npm run release-check
git diff --check
```

When provider environment variables are absent, release checks must keep:

```text
provider_probe_status=skipped
```
