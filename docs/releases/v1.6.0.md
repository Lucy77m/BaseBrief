## Fixture Contract

The fixture pack keeps the v1.5 receiver lint contract stable:

- clean receiver fixtures must pass with zero errors and zero warnings
- error fixtures are intentionally broken and must fail with the documented
  primary receiver rule family
- warning fixtures must keep exit-success checker status while reporting the
  documented warning
- `difference_found` remains a completed verification result, not an agent
  failure
- historical `commits_in_range` drift remains non-blocking when refreshed
  branch, HEAD, and worktree facts still match live repository state
