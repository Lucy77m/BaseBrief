# v0.9.2 Golden Path Example Closure Candidate

`v0.9.2` closes the public example gap after `v0.9.1` Golden Path Closure.
It does not define a new command, schema, or runtime line. It adds a
public-safe example kit so users can follow the integrated local handoff path
with something concrete in hand.

## Example Goal

The target workflow remains:

```text
receiver-ready.md -> state-init/state-advance -> sidecar-build -> sidecar-check -> new-window-starter.md -> receiver first response
```

This candidate keeps both branches visible in public examples:

- first-pass uses `state-init`
- follow-up uses `state-advance`
- both branches stop at the receiver first-response gate before project work

## What Changed

- Adds `examples/golden-path/` as a public-safe example kit for the integrated
  local handoff line.
- Links the example kit from the golden-path doc, quickstart, README, docs
  index, and usability feedback template.
- Keeps raw Sidecar output in ignored private directories while still showing
  reviewed handoff shape, Project State shape reference, and receiver
  first-response examples.
- Adds release-check and test coverage so example discoverability and public
  safety stay enforced locally.

## Boundaries

- No provider request.
- No raw private output.
- No runtime integration.
- No schema change.
- No Auto Flow.
- No plugin or platform work.
- No published npm package or global CLI.
- No v1.0 claim.
