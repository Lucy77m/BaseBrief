# v2.2.0 One-command Resume / New-window Prompt Plan

Date: 2026-06-08

Status: v2.2-A contract freeze

This document freezes the v2.2 One-command Resume / New-window Prompt line.
It is docs-first and local-only. It does not reopen the v2.1 checker line, does
not change Context Pack Lite generator output, and does not change the existing
`check --input <dir> --json` top-level shape.

## Goal

BaseBrief v2.2 adds a narrow resume surface:

```text
node scripts/basebrief.js resume --input <context-pack-dir> [--json]
```

The command reads an already generated Context Pack Lite directory, runs the
existing artifact checker against it, and prints a copyable new-window prompt.
The prompt tells the receiver to read the pack, recheck live repo facts, report
gaps, and preserve risk boundaries before implementation.

## Contract

The v2.2-A contract is:

- input is a Context Pack Lite directory with the seven v2.0 files
- validation reuses the existing Artifact Checker behavior from v2.1
- clean packs produce a copyable prompt
- warning-only packs still produce a prompt and include the warning findings
- error findings stop the command before a continuation prompt is printed
- human output is the prompt text
- JSON output includes `command`, `contractVersion`, `input`, `status`,
  `prompt`, `promptLength`, and the existing checker result nested under
  `check`

The command should print prompt text only. It should not create external
sessions, call providers, call AI, run a daemon, watch files, or modify files.

## Non-goals

- no provider request
- no runtime integration
- no plugin, MCP, IDE, hosted service, or cloud-memory behavior
- no schema-v2, `basebrief-project-state-v2`, or `basebrief-sidecar-v2`
- No Workflow Runner
- No context-pack generator output change
- No `check --input <dir> --json` top-level shape change
- no `context-pack-check` command
- no `new-window` alias in the first slice
- no output file path in the first slice
- no push, tag, release, pull request, npm publish, or global CLI install
- no raw private output, `.env`, secrets, tokens, credentials, or API keys

## Implementation Slice

The smallest acceptable v2.2-B implementation is:

- `scripts/basebrief_resume.js` for read-only prompt assembly
- `scripts/basebrief.js` command routing and help text
- independent tests for clean, warning-only, and error input behavior
- release-check assertions for docs, CLI help, and command behavior
- docs discoverability in CLI, testing, roadmap, and index docs

## Validation Gate

The local validation gate is:

```text
node --test tests/basebrief.test.js --test-name-pattern "resume"
node scripts/basebrief.js resume --input examples/context-pack-lite --json
node scripts/basebrief.js check --input examples/context-pack-lite --json
npm test
npm run release-check
git diff --check
```

When provider environment variables are absent, release checks must keep:

```text
provider_probe_status=skipped
```

## Closeout Criteria

The line can move from v2.2-A to v2.2-B when:

- this plan and `docs/specs/context-pack-resume.md` are discoverable
- `resume --input <context-pack-dir>` is implemented as a read-only command
- clean and warning-only Context Pack inputs produce copyable prompts
- error Context Pack inputs are rejected before prompt output
- v2.1 checker and v2.0 generator contracts remain unchanged
