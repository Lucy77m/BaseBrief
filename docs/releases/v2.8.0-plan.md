# v2.8.0 Continuation Harness Lite Plan

Date: 2026-06-10

Status: v2.8-A continuation harness planning

This document opens the v2.8 line after the v2.7.1 Context Pack human
next-step hints release. v2.8 moves BaseBrief from polishing individual
Context Pack commands toward assembling the existing commands into one
reviewable local continuation package.

It adds one thin local CLI surface, `continue`, but does not create a Workflow
Runner, provider integration, runtime integration, hosted service, plugin, MCP
server, schema-v2, watcher, daemon, automatic git action, or task executor.

## Goal

Continuation Harness Lite prepares a receiver-ready local package by running
the proven Context Pack path in sequence:

```text
context-pack -> check -> resume
```

The command should make the handoff flow easier to repeat without hiding the
underlying files or turning BaseBrief into automation. The output must remain
reviewable by a human before it is copied into a new AI window.

## Command Shape

The v2.8-A command shape is:

```text
node scripts/basebrief.js continue --repo <target-repo> --output-dir <dir> [--since <commit>] [--max-files <n>] [--json]
```

The command writes a local continuation package under the explicit output
directory. The output directory must be empty or absent and must not use
`.env` or `.git` paths.

## Minimal Output

The first implementation writes:

```text
CONTINUATION_REPORT.md
NEXT_WINDOW_STARTER.md
CHECK_SUMMARY.md
continuation.meta.json
context-pack/
```

`context-pack/` keeps the existing seven-file Context Pack Lite shape:

```text
MANIFEST.md
REPO_MAP.md
KEY_FILES.md
RECENT_DELTA.md
RISK_BOUNDARIES.md
RECEIVER_STATE.md
NEXT_WINDOW_STARTER.md
```

The top-level `NEXT_WINDOW_STARTER.md` is copied from `resume` output with the
private generated pack path normalized to `context-pack`. It must not expose
private absolute paths.

## Status Semantics

The command reports:

- `ready_for_receiver`: clean Context Pack check, clean worktree, and resume
  prompt ready
- `needs_review`: warning-only Context Pack check or dirty worktree, with
  resume prompt ready
- `blocked`: Context Pack check errors or resume generation failure

`blocked` exits nonzero. `ready_for_receiver` and `needs_review` exit zero.

## JSON Contract

`continue --json` returns stable command metadata, output file paths, step
status, git summary, and check summary. It must not include the full resume
prompt body or human-only `next_step` fields.

Existing Context Pack Check, Resume, Export, and Doctor JSON contracts remain
unchanged.

## Boundaries

- No provider request.
- No runtime integration.
- No plugin.
- No MCP server.
- No MCP tools.
- No schema-v2.
- No Workflow Runner.
- No Doctor expansion.
- No Export expansion.
- No automatic project task implementation.
- No automatic commit, push, tag, release, pull request, npm publish, or global
  CLI install.
- No package script other than extending the existing local `npm test` file
  list.
- No Context Pack seven-file structure change.
- No raw private output, private absolute paths, `.env`, secrets, tokens,
  credentials, API keys, or bearer strings.

## Validation Matrix

The v2.8 implementation must verify:

```text
clean repo -> continue -> ready_for_receiver
dirty repo -> continue -> needs_review
unsafe output path -> rejected
non-empty output directory -> rejected
continue --json -> public-safe metadata only
copied starter -> no private absolute context-pack path
```

The local validation gate is:

```text
node --test tests/continuation-harness.test.js
npm run release-check
npm test
git diff --check
```

When provider environment variables are absent, release checks must keep:

```text
provider_probe_status=skipped
```

## Rollback

If the implementation starts requiring provider calls, schema-v2, Workflow
Runner behavior, Doctor expansion, runtime integration, or automatic git
actions, stop and keep v2.8 as a plan only. A valid rollback removes the thin
`continue` command, its docs, tests, example kit, dogfooding record, and
release-check assertions without touching the underlying Context Pack commands.
