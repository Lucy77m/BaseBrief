## Repair Contract

The repair pack keeps the v1.5 and v1.6 contracts stable:

- missing Delta sections are repaired by restoring the required report section,
  not by changing detection
- missing starter anchors are repaired by restoring `pass/fail` and
  `wait for user confirmation`
- missing fact layers are repaired by separating source-window inherited facts,
  live repo facts, and receiver-window rechecks
- `receiver.invalid-result-consistency` is repaired by aligning
  `receiver_task_status`, `repository_state_status`, and
  `handoff_acceptance`
- `difference_found` remains a completed verification result, not an agent
  failure
- historical `commits_in_range` drift remains non-blocking when refreshed
  branch, HEAD, and worktree facts still match live repository state
