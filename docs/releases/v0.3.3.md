# BaseBrief v0.3.3 Release Candidate

Status: local release candidate

Date: 2026-06-05

BaseBrief v0.3.3 adds public-safe dogfooding evidence for the v0.3.2 Receiver Flow Draft Skeleton. It is evidence-only stabilization: no new command, no schema change, no provider request, and no Auto Flow.

## Highlights

### Receiver Flow Dogfooding Evidence

- Added a public-safe receiver-flow dogfooding record.
- Added examples for clean repo drafts, dirty repo drafts, and Git-visible output drafts.
- Kept every example in review state:

```text
handoff_status: draft_needs_review
```

- Repeated the manual review boundary before any draft can be rewritten into a receiver-ready handoff.

## Compatibility

- BB9 handoff schema is unchanged.
- Receiver Safe Check config and result schemas are unchanged.
- CLI Lite command behavior is unchanged.
- `receiver-flow --repo <target-repo> --output-dir <dir> [--json]` is unchanged.
- No npm scripts or package metadata are added.
- No Auto Flow is introduced.

## Evidence

Repository-reproducible validation for this release candidate includes:

- `git diff --check`
