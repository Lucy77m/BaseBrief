## Report Contract

The v1.2 report kit keeps these receiver report fields stable:

- `receiver_task_status`
- `repository_state_status`
- `handoff_acceptance`
- `blocking_or_repair_notes`
- `current_goal`
- `live_repo_state`
- `inherited_fact_differences`
- `hard_boundaries`
- `next_narrow_slice`

The kit keeps source-window inherited facts separate from receiver-window
rechecks before the receiver recommends the next narrow slice.

## Acceptance Semantics

- `handoff_acceptance: pass` means the entry verification completed and live
  branch, HEAD, and worktree facts matched the refreshed handoff facts.
- `handoff_acceptance: difference_found` means the entry verification completed
  and accurately reported a live-state difference. It is not an agent failure.
- `handoff_acceptance: blocked` means the receiver could not complete necessary
  verification safely.

Historical dry-run or pre-commit `commits_in_range` values remain historical
evidence. Explainable historical count drift is non-blocking when refreshed
`delta-handoff.md` branch, HEAD, and worktree facts match live repository state.
