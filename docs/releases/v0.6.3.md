# v0.6.3 Lifecycle Readiness Gate Candidate

`v0.6.3` is a readiness-gate candidate for future Project State Lifecycle work.
It turns `v0.6.2` self-dogfooding friction into public criteria for when
lifecycle commands may be planned.

This is not a lifecycle release.

## Delivered

- Project State Lifecycle readiness design boundary:
  `docs/design/project-state-lifecycle-readiness.md`.
- Public-safe dogfooding record:
  `docs/dogfooding/project-state-lifecycle-readiness-v0.6.3.md`.
- `v0.6.x` matrix entries that classify friction as testable, documentable, or
  not automated yet.
- Release-check and test contracts for the readiness gate and Artifact Checker
  safety pass.

## Decision

Duplicate state, missing state, and review-gate friction are lifecycle design
signals. They are not enough to implement automatic state advancement or status
inference.

Future lifecycle work must pass the readiness criteria before adding commands.

## Boundaries

- No state lifecycle commands.
- No Auto Flow.
- No provider request.
- No schema change.
- No provider gateway.
- No sidecar adapter.
- No published npm package.
- No global CLI installation.
- No push, tag, or release action in this implementation pass.
- `provider_probe_status=skipped` remains the expected no-provider gate.

## Provider Matrix Position

Provider smoke remains a private matrix design only. Public docs may mention the
environment variable names below, but must not record endpoint values, keys,
model values, raw provider responses, or provider claims as release proof:

```text
BASEBRIEF_PROVIDER_BASE_URL
BASEBRIEF_PROVIDER_API_KEY
BASEBRIEF_PROVIDER_MODEL
```
