# BaseBrief v0.5.0 Guided Receiver Flow Candidate

Date: 2026-06-05

`v0.5.0` adds `receiver-flow --guided`, a human-input mode that makes receiver-flow
drafts less empty while keeping the draft-only safety boundary.

## Delivered Scope

- Added `node scripts/basebrief.js receiver-flow --repo <target-repo> --output-dir <dir> --guided [--json]`.
- Default `receiver-flow` behavior remains unchanged when `--guided` is absent.
- Guided mode accepts six human-provided fields: `current_goal`, `verified_facts`, `confirmed_decisions`, `risk_boundaries`, `receiver_entry_task`, and `open_questions`.
- Empty guided answers are written as `[EMPTY]`.
- `draft-context.md` includes `Human-Provided Fields`, `Review Checklist`, and guided safety notes.
- `flow-summary.json` includes `guided`, `human_fields`, and `review_checklist` fields for guided runs.
- `handoff_status: draft_needs_review` remains fixed.

## Compatibility

- BB9 handoff schema is unchanged.
- Receiver Safe Check config and result schemas are unchanged.
- Default Receiver Flow Draft output remains draft-only.
- CLI Lite command family remains local and zero-dependency.
- `package.json` remains private and validation-only.
- `provider_probe_status=skipped` remains the expected release-check result when provider env vars are absent.
