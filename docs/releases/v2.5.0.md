# v2.5.0 Context Pack Doctor Local Closeout

Date: 2026-06-08

Status: local closeout

`v2.5.0` closes the first Context Pack Doctor implementation slice. It adds a
local-only, read-only diagnostic command:

```text
node scripts/basebrief.js doctor --repo <target-repo> --context-pack <context-pack-dir> [--json]
```

## What Changed

- Adds `scripts/basebrief_doctor.js` as the doctor module.
- Adds CLI Lite `doctor` wiring through `scripts/basebrief.js`.
- Adds the JSON contract version `basebrief-doctor-v1`.
- Reports findings with fixed severity, rule id, source, message, and
  public-safe evidence fields.
- Reuses Context Pack Check and propagates checker errors as doctor errors.
- Adds stale HEAD, branch mismatch, dirty worktree, live recheck, and boundary
  wording diagnostics.
- Adds `examples/context-pack-doctor/` as a public-safe sample output kit.

## Boundaries

- No `status` command in v2.5.
- No watcher, daemon, background monitor, or auto-fix.
- No Context Pack Lite generator output change.
- No `check --input <dir> --json` top-level shape change.
- No `resume --input <context-pack-dir>` behavior change.
- No `export --input <context-pack-dir> --output-dir <dir>` contract change.
- No provider request.
- No runtime integration.
- No plugin.
- No MCP server.
- No MCP tools.
- No IDE integration.
- No hosted service.
- No cloud-memory behavior.
- No schema-v2.
- No Workflow Runner.
- No push, tag, release, pull request, npm publish, or global CLI install.

Doctor is a snapshot diagnostic. A receiver must still recheck cwd, branch,
HEAD, worktree status, and relevant files before implementation.
