# v2.4.0 File-only Adapter / MCP-friendly Export Local Closeout

Date: 2026-06-08

Status: local closeout

`v2.4.0` closes the first File-only Adapter / MCP-friendly Export
implementation slice. It adds a local-only export command for checked Context
Pack Lite directories:

```text
node scripts/basebrief.js export --input <context-pack-dir> --output-dir <dir> [--json]
```

## What Changed

- Adds `scripts/basebrief_export.js` as the file-only export module.
- Adds CLI Lite `export` wiring through `scripts/basebrief.js`.
- Writes exactly four export files:
  - `manifest.json`
  - `context-pack.md`
  - `context.json`
  - `adapter-notes.md`
- Allows warning-only Context Pack inputs and records warnings in the export.
- Blocks errored Context Pack inputs before writing export files.
- Adds `examples/file-only-export/` as a public-safe example kit.
- Clarifies that `exports/` is a recommended example output directory name;
  the CLI writes directly under explicit `--output-dir`.

## Boundaries

- No Context Pack Lite generator output change.
- No `check --input <dir> --json` top-level shape change.
- No `resume --input <context-pack-dir>` behavior change.
- No provider request.
- No runtime integration.
- No plugin.
- No MCP server.
- No IDE integration.
- No hosted service.
- No cloud-memory behavior.
- No schema-v2.
- No Workflow Runner.
- No push, tag, release, pull request, npm publish, or global CLI install.

MCP-friendly means local files are easier for future tools to consume. It does
not mean BaseBrief creates an MCP server, registers tools, opens a transport,
manages sessions, or writes host-tool configuration.

## Validation Gate

The local validation gate for this closeout is:

```text

node scripts/basebrief.js export --input examples/context-pack-lite --output-dir tests/outputs/private/file-export --json
```
