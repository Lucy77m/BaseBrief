# v1.7.0 Delta Receiver Lint Repair Pack Plan

Date: 2026-06-07

This document records the recommended v1.7.0 planning direction after the
v1.6 Delta Receiver Lint Fixture Pack line became locally closed.

It is a planning baseline, not a push, tag, release, package publication,
provider integration, runtime integration, plugin, MCP, IDE, hosted service,
Auto Flow behavior, global CLI install, npm publication, command output format
change, JSON schema claim, or schema-v2 claim.

## Summary

BaseBrief v1.7.0 should keep the receiver lint line local-first and
review-based by adding a repair guide:

```text
Delta Receiver Lint Repair Pack
```

The goal is not to add checker rules or change command output. The goal is to
show authors how to repair each v1.5 receiver lint rule family after they learn
the failures from the v1.6 fixture pack.

## Recommended Direction

Prefer a docs/examples/tests slice:

- add `examples/receiver/lint/repair/README.md` as the public repair guide
- map receiver lint rule IDs to the smallest repair action
- add fixed public-safe Delta Markdown, starter Markdown, and receiver result
  JSON examples
- verify the fixed examples pass with zero errors and zero warnings
- keep intentionally broken fixtures in `examples/receiver/lint/` unchanged

## Repair Coverage

The repair pack should cover:

- `receiver.missing-report-section`
- `receiver.missing-human-anchor`
- `receiver.missing-fact-layer`
- `receiver.invalid-result-consistency`
- `receiver.missing-difference-semantics`
- `receiver.missing-drift-semantics`

## v1.7.0 In Scope

- Public-safe repair guide and fixed examples.
- Documentation updates that route from receiver lint fixtures to repair
  guidance.
- Release-check and test coverage for fixed examples.

## v1.7.0 Out Of Scope

- No provider request.
- No runtime integration.
- No plugin, MCP, IDE, hosted, or cloud-memory work.
- No schema-v2 work.
- No `basebrief-project-state-v2`.
- No `basebrief-sidecar-v2`.
- No new CLI command.
- No new npm script.
- No machine-readable JSON schema.
- No command output format change.
- No Auto Flow.
- No checker rule change.
- No push, tag, release, pull request, npm publish, or global CLI install.
- No raw private output in public docs.

## Success Criteria

v1.7.0 is successful if all of the following are true:

1. The repair guide maps each receiver lint rule family to a concrete fix.
2. The fixed Delta Markdown example passes Artifact Checker with zero findings.
3. The fixed starter Markdown example passes Artifact Checker with zero
   findings.
4. The fixed receiver result JSON example passes Artifact Checker with zero
   findings.
5. No schema, command, runtime, provider, plugin, MCP, IDE, Auto Flow, hosted,
   checker-rule, or publication scope is introduced.

When provider environment variables are absent, release checks must keep:

```text
provider_probe_status=skipped
```
