# v0.6.2 Self-Dogfooding Evidence Candidate

`v0.6.2` is an evidence/stabilization candidate for BaseBrief's project-state
self-dogfooding line. It records normal and exception-path evidence without
adding CLI commands, changing schemas, or treating provider smoke as public
release proof.

## Delivered

- Public-safe dogfooding record:
  `docs/dogfooding/project-state-self-dogfooding-v0.6.2.md`.
- `v0.6.x` test matrix coverage for reviewed state creation and rejected
  project-state inputs.
- Receiver friction log entries for review-gate and state-init friction.
- Release-check and test contracts for the new documentation and Artifact
  Checker safety pass.

## Evidence Scope

- `receiver-flow --guided` creates `handoff_status: draft_needs_review`.
- `review-draft` promotes only explicitly reviewed content to
  `handoff_status: ready_for_receiver`.
- `state-init` accepts the reviewed source and writes
  `basebrief-project-state-v1`.
- `state-read --json` reads the local state back.
- Exception evidence covers `review-draft-unchecked`,
  `state-init-draft-rejected`, `state-read-missing-state`,
  `state-init-env-source-rejected`, `state-init-git-source-rejected`,
  `state-init-missing-field-rejected`, and `state-init-duplicate-rejected`.

## Boundaries

- No provider request.
- No Auto Flow.
- No state lifecycle commands.
- No schema change.
- No sidecar adapter.
- No provider gateway.
- No published npm package.
- No global CLI installation.
- No push, tag, or release action in this implementation pass.
- `provider_probe_status=skipped` remains the expected no-provider gate.

## Provider Matrix Position

Provider smoke remains a private matrix design only. Public docs may mention the
environment variable names below, but must not record endpoint values, keys,
model values, raw provider responses, or provider claims as release proof:

```text
BASEBRIEF_PROVIDER_BASE_URL
BASEBRIEF_PROVIDER_API_KEY
BASEBRIEF_PROVIDER_MODEL
```
