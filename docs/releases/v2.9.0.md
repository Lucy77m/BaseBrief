# v2.9.0 Project Profile / Recipes Lite Local Closeout

Date: 2026-06-10

Status: local closeout

`v2.9.0` implements Project Profile / Recipes Lite as a narrow companion to
Continuation Harness Lite. It adds a local profile template command and lets
`continue` read public-safe recipe defaults:

```text
node scripts/basebrief.js profile-init --repo <target-repo> --output <profile.json> [--recipe continuation-default|small-delta|review-heavy] [--json]
node scripts/basebrief.js continue --profile <profile.json> --output-dir <dir> [--repo <target-repo>] [--since <commit>] [--max-files <n>] [--json]
```

## What Changed

- Adds `scripts/basebrief_project_profile.js` with
  `basebrief-project-profile-v1` validation.
- Exposes `profile-init` through CLI Lite help, human output, and JSON output.
- Extends `continue` with `--profile` while preserving the v2.8
  `continue --repo` path.
- Adds three recipes: `continuation-default`, `small-delta`, and
  `review-heavy`.
- Lets explicit CLI flags override profile defaults for repo, `since`, and
  `max-files`.
- Adds independent tests in `tests/project-profile.test.js`.
- Adds the Project Profile Lite example kit under
  `examples/project-profile-lite/`.

## JSON Boundary

`profile-init --json` reports profile metadata and the generated public-safe
profile body. `continue --profile --json` reports metadata, output file paths,
profile path, recipe, step status, git summary, and check summary. It does not
include the full resume prompt body and does not add human-only `next_step`
fields.

Existing Context Pack Check, Resume, Export, Doctor, and plain `continue --repo`
JSON contracts remain unchanged.
