# v2.6.0 First-Run / Adoption Polish Local Closeout

Date: 2026-06-09

Status: local closeout

`v2.6.0` closes the First-Run / Adoption Polish slice. It does not add a new
command, JSON contract, schema, provider path, runtime integration, plugin, MCP
surface, Workflow Runner, or always-on status feature.

## What Changed

- Adds a first-run kit to the quickstart so a new user can complete a
  2-5 minute local-first loop before reading historical release docs.
- Expands the minimal example with a short read order, expected loop, and
  receiver acceptance semantics.
- Adds adoption examples for clean, warning, and broken Context Pack inputs.
- Clarifies Check vs Doctor:
  - `check` answers whether a Context Pack is structurally reviewable.
  - `doctor` answers whether a specific pack still matches live repository
    facts.
- Documents how to interpret `context-pack.too-thick`,
  `doctor.pack-head-stale`, and `doctor.pack-check-error`.
- Reworks `docs/index.md` as a documentation map with first-run, continuation,
  diagnostics, advanced reference, and archive sections.
- Adds release-check assertions to keep the adoption path discoverable and to
  keep historical archives out of the first-run path.
