# BaseBrief v0.7.0 Project State Lifecycle Candidate

`v0.7.0` is the Project State Lifecycle candidate. It adds explicit local
lifecycle commands for the existing `.basebrief/state.json` model while keeping
the `basebrief-project-state-v1` schema unchanged.

## Added

- `state-status --repo <target-repo>`
- `state-validate --repo <target-repo>`
- `state-history --repo <target-repo>`
- `state-advance --repo <target-repo> --source <receiver-ready.md>`

`state-advance` archives the previous state under `.basebrief/history/` before
writing the next reviewed state to `.basebrief/state.json`.

## Preserved

- `state-init` behavior is unchanged.
- `state-read` behavior remains read-only.
- `basebrief-project-state-v1` is unchanged.
- Receiver Safe Check schemas are unchanged.
- BB9 handoff schema is unchanged.
- `review-draft` remains the required gate before a source can be used for
  `state-init` or `state-advance`.

## Verification Summary

- Unit tests cover missing state, invalid state, reviewed advancement, history
  archive creation, draft rejection, `.env` path rejection, and `.git` path
  rejection.
- Release checks run a private local CLI smoke through
  `receiver-flow --guided -> review-draft -> state-init -> state-read ->
  state-status -> state-validate -> state-history -> state-advance ->
  state-history`.
- Artifact Checker covers the new public docs.
- Provider probing remains skipped by default:
  `provider_probe_status=skipped`.

## Boundaries

- No Auto Flow.
- No provider request.
- No schema change.
- No receiver thread creation.
- No provider gateway.
- No published npm package.
- No global CLI installation.
- No push, tag, or formal release in the candidate commit.

Provider private validation remains future work and may only be represented in
public material by env var names: `BASEBRIEF_PROVIDER_BASE_URL`,
`BASEBRIEF_PROVIDER_API_KEY`, and `BASEBRIEF_PROVIDER_MODEL`.
