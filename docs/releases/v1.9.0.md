## Adoption Contract

The public receiver lint adoption path is:

1. Choose the receiver route in `docs/receiver-usage-pack.md`.
2. Pick the closest public example in
   `examples/receiver/usage-pack/README.md`.
3. Use `examples/receiver/lint/README.md` only to learn a rule family or
   intentionally broken shape.
4. Use `examples/receiver/lint/repair/README.md` for the smallest fixed Delta,
   starter, or result JSON reference.
5. Copy the final report shape from existing receiver examples such as
   `examples/receiver/delta-report-pass/README.md`,
   `examples/receiver/delta-report-difference-found/README.md`,
   `examples/receiver/blocked/README.md`,
   `examples/receiver/language-routing/README.md`, or the golden-path starter
   reports.

Intentional failing fixtures remain learning inputs, not handoff copy targets.

## Contract Boundaries

- `difference_found` remains a completed verification result, not an agent

- Historical `commits_in_range` drift remains non-blocking when refreshed
  branch, HEAD, and worktree facts still match live repository state.
- Receiver lint remains explicit-shape based; it does not become a whole-repo
  keyword scan.
