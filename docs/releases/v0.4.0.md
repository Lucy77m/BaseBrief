# BaseBrief v0.4.0 Release Candidate

Status: local release candidate

Date: 2026-06-05

BaseBrief v0.4.0 is the integrated local toolchain release candidate. It collects the v0.3.x receiver work into the public release line while keeping BaseBrief a local handoff toolchain: one public skill entry, BB9 handoff artifacts, CLI Lite, Artifact Checker, Receiver Safe Check, Receiver Flow Draft, Seal/Diff, local npm validation scripts, and public-safe evidence.

This is not a hosted platform, not a published npm package, not an installed global CLI, not a provider gateway, and not Auto Flow.

## Integrated Toolchain

- One public BaseBrief skill entry with internal Full / Lite routing.
- `cache-ready` remains an explicit prompt-cache experiment route, not a normal third continuation mode.
- BB9 structured handoff contract and generated handoff artifacts.
- File-based Codex and Claude adapter outputs.
- Artifact Checker for public-safe local output review.
- Receiver Safe Check v1 and reviewed Receiver Flow Draft.
- Seal/Diff v1 for phase comparison.
- Minimal local npm validation scripts:
  - `npm test`
  - `npm run release-check`
  - `npm run check`

## Compatibility

- BB9 handoff schema is unchanged.
- Receiver Safe Check config and result schemas are unchanged.
- Seal schema is unchanged.
- CLI Lite command behavior is unchanged.
- `package.json` stays private and validation-only.
- Provider profiles and cache-ready evidence wording remain provider-specific and conservative.

## Evidence

Repository-reproducible validation for this release candidate includes:

- `git diff --check`
- `node --test tests/basebrief.test.js`
- `node scripts/run_release_checks.js`
- `npm run check`

The release check must preserve `provider_probe_status=skipped` when provider environment variables are absent.

No provider request is required or performed for this release-candidate preparation. No receiver thread creation, external receiver matrix, OpenCode smoke, Claude Code smoke, Auto Flow run, Web UI check, Cursor adapter test, or provider benchmark is required for v0.4.0.

## Release Gate

The local release candidate is ready only after:

- all automated tests and release checks pass
- public artifacts report zero Artifact Checker errors and warnings
- protected interfaces remain unchanged
- commits contain no secrets or private absolute paths
- `provider_probe_status=skipped` is reported when provider env vars are absent
- the user separately approves push, tag, and release

This document is not a release announcement. No push, tag, or formal release is performed by the release-candidate preparation task.
