# BaseBrief v0.4.1 Stabilization Candidate

Date: 2026-06-05

`v0.4.1` is a stabilization candidate for the already published `v0.4.0` local
toolchain. It adds public-safe validation records, stronger local safety checks, and
edge-case coverage without changing the product surface.

## Delivered Scope

- Added a post-release baseline for `v0.4.0`.
- Added a v0.4.x test matrix for regression, edge-case, security, scale, and runner dogfooding work.
- Strengthened Artifact Checker coverage for common fake secret shapes.
- Added tests for Unicode and space paths, no-git receiver-flow failure, broken Markdown, skipped noisy directories, and fake secret blocking.
- Kept external runner evidence public-safe and summary-only.

## Compatibility

- BB9 handoff schema is unchanged.
- Receiver Safe Check config and result schemas are unchanged.
- Receiver Flow Draft schema and default behavior are unchanged.
- CLI Lite command behavior is unchanged.
- `package.json` remains private and validation-only.
- `provider_probe_status=skipped` remains the expected release-check result when provider env vars are absent.

## Non-Goals

- No provider request.
- No broad provider savings claim.
- No live cost audit.
- No receiver thread creation.
- No Auto Flow.
- No `receiver-flow --guided`.
- No `receiver-flow --extract`.
- No `review-draft`.
- No `.basebrief/` project state directory.
- No Web UI.
- No Cursor, Cline, Roo, or OpenCode adapter expansion.
- No hosted service.
- No installed or global CLI.
- No published npm package.
- No push, tag, or formal release by this implementation step.

## Verification Gate

The candidate is ready only when these local checks pass:

```text
git diff --check
node --test tests/basebrief.test.js
node scripts/run_release_checks.js
npm run check
```
