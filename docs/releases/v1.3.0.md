## Summary

BaseBrief v1.3 closes one narrow local-first capability:

```text
Delta Receiver Starter Integration
```

The line connects the v1.2 Delta Receiver Report Kit to starter-facing docs,
golden-path examples, and the copyable receiver starter contract. A receiver no
longer needs to invent a starter report shape after reading `quickstart`,
`golden-path`, or `next-chat-prompt`.

## Starter Contract

The v1.3 starter integration keeps these human-facing anchors stable:

- human-facing `pass/fail`
- `wait for user confirmation`
- `declared_checks_status` when Receiver Safe Check is used
- existing receiver-ready protocol literals and fixed command shapes

Starter-facing reports may now also reuse the v1.2 report-kit fields:

- `current_goal`
- `live_repo_state`
- `inherited_fact_differences`
- `hard_boundaries`
- `next_narrow_slice`

The starter line keeps source-window inherited facts separate from live repo
facts and receiver-window rechecks before implementation resumes.

## Acceptance Semantics

- human-facing `fail` may coexist with
  `handoff_acceptance: difference_found`; that still means the receiver
  completed entry verification and accurately reported a live-state difference
- `difference_found` is not an agent execution failure
- historical dry-run or pre-commit `commits_in_range` values remain historical
  evidence; explainable historical count drift is non-blocking when refreshed
  branch, HEAD, and worktree facts match live repository state
