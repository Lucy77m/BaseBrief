# v0.8.3 Sidecar Discoverability Polish Candidate

`v0.8.3` is a public documentation and release-readiness polish pass for the
v0.8.x Sidecar work. It does not add a new CLI command, API, schema, provider
smoke, runtime integration, tag, release, or npm publish step.

## What Changed

- README and README.en now expose the Project State Sidecar path.
- The public docs index links the v0.8.0, v0.8.1, v0.8.2, and v0.8.3 records.
- The v0.8.x test matrix records Sidecar discoverability coverage.
- Release checks assert README/docs mention `sidecar-build`, `sidecar-check`,
  `generic`, `openclaw`, and `provider_probe_status=skipped`.

## Boundaries

- No provider request.
- No raw private output.
- No runtime integration.
- No schema change.
- No Auto Flow.
- No receiver thread creation.
- Wait for user confirmation before continuing from a generated bundle.
- No OpenClaw/Hermes runtime connection.
- No OpenClaw/Hermes profile/config/memory/workspace writes.
- No push, tag, release, or npm publish in this checkpoint.

`sidecar-build` remains a local Project State consumer for
`basebrief-project-state-v1`. `sidecar-check` remains a read-only structure and
public-safety gate for `basebrief-sidecar-v1` bundles.

`provider_probe_status=skipped`
