# BaseBrief v0.4.0 Post-Release Baseline

Date: 2026-06-05

This baseline records the public-safe state after the maintainer published `v0.4.0`.
It is a planning and validation anchor for the v0.4.x stabilization cycle, not a new
feature announcement.

## Release Anchor

- release tag: `v0.4.0`
- release commit: `4de7342`
- public status: published by maintainer before this stabilization cycle
- local source state expected for v0.4.x work: clean checkout at the release tag or a later local stabilization branch

## Stable Capabilities

- One public BaseBrief skill entry with internal Full / Lite routing.
- BB9 structured handoff contract.
- CLI Lite commands: `init`, `build`, `check`, `receiver-init`, `receiver-check`, `receiver-flow`, `seal`, and `diff`.
- Artifact Checker for public-safe output review.
- Receiver Safe Check v1 with `pass`, `difference_found`, and `blocked` semantics.
- Receiver Flow Draft with `handoff_status: draft_needs_review`.
- Seal/Diff v1 for file-based phase comparison.
- Local npm validation scripts: `npm test`, `npm run release-check`, and `npm run check`.

## v0.4.x Stabilization Target

The v0.4.x cycle should validate the existing toolchain before adding new workflow
features. The cycle may add tests, public-safe evidence, checker rules, documentation
corrections, and small bug fixes.

The cycle should not add `receiver-flow --guided`, `receiver-flow --extract`,
`review-draft`, `.basebrief/`, Auto Flow, Web UI, adapter expansion, provider
benchmark claims, or npm publishing.

## Validation Boundaries

- No provider request.
- No broad provider savings claim.
- No live cost audit.
- No external runner raw output in public docs.
- No API key, token, secret, `.env` content, or private absolute path in public artifacts.
- `provider_probe_status=skipped` remains the expected release-check status when provider env vars are absent.
- No push, tag, or formal release is performed by the stabilization implementation itself.
