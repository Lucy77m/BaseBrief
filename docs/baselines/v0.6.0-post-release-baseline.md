# BaseBrief v0.6.0 Post-Release Baseline

Date: 2026-06-05

`v0.6.0` is the first published BaseBrief release with a local project-state
directory. The release combines the `v0.5.x` receiver-flow stabilization line
with the initial `.basebrief/state.json` continuity surface.

## Baseline Facts

- release tag: `v0.6.0`
- release commit: `417541a`
- release status: formal GitHub release
- local project state schema: `basebrief-project-state-v1`
- project-state commands: `state-init`, `state-read`
- provider probe in normal local checks: `provider_probe_status=skipped`

## Public Positioning

BaseBrief remains a local handoff toolchain. It is not Auto Flow, not a hosted
platform, not a provider gateway, not a published npm package, and not a global
CLI.

`state-init` creates local state only from a reviewed source with:

```text
handoff_status: ready_for_receiver
```

It does not promote a draft and does not confirm candidate facts.

## Next Stabilization Target

`v0.6.1` should tighten the state model, validation wording, v0.6.x test
matrix, and BaseBrief self-dogfooding evidence before any `v0.7.0` Project
State Lifecycle work begins.

## Non-Goals

- No Auto Flow implementation.
- No `state-advance`, `state-status`, `state-validate`, or `state-history`.
- No sidecar adapter or external tool adapter.
- No provider request or public provider claim.
- No schema change to `basebrief-project-state-v1`.
