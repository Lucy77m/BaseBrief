# BaseBrief v0.2.2 First-run Workflow Dogfooding

Date: 2026-06-04

This record documents one public-safe self-dogfooding pass over the first-run workflow. It is evidence about the local BaseBrief repository workflow, not a claim about external adoption.

## Workflow

The pass followed the public Quickstart:

1. Build the structured Lite example with `--check`.
2. Check the generated output directory directly.
3. Create a seal from the public before fixture.
4. Diff that seal against the public after fixture.

All generated files stayed under `tests/outputs/private/quickstart/`, which is ignored by the repository.

## Finding

The first pass completed successfully but reported two `artifact.missing-open-questions` warnings.

The structured Lite example did not include `open_questions`, and the Lite readable renderer did not emit the section even when input data provided it. The CLI human-readable summary reported the warning count but did not show the finding details.

## Resolution

- The structured Lite example now includes one public-safe open question.
- Full and Lite readable handoffs both preserve `open_questions`; `assumptions` remains Full-only.
- CLI human-readable `build --check` and `check` output now lists each finding.
- Release checks execute the public first-run workflow and require zero errors and zero warnings.

## Remaining Friction

- Users must provide structured JSON or marked structured Markdown; free-form Markdown is not parsed.
- CLI commands require explicit input and output paths.
- Seal/Diff compares BaseBrief handoff fields, not arbitrary repository changes.
- The artifact checker is a narrow deterministic check, not a complete security review.

See [Known Limitations](../known-limitations.md) for the maintained boundary list.
